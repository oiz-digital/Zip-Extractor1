## Mainnet Blockers (originally 6 — 3 fixed, 3 remaining)

### MB-1: KZG Trusted Setup Ceremony File Missing
**File:** `zbx-da/src/commitment.rs:113-146`  
Without `/etc/zbx/kzg_g2_tau.bin` (or `ZBX_KZG_G2_TAU_PATH`), the node panics on `ZBX_CHAIN_ENV=mainnet`. This is correct behavior — but the ceremony file itself must be generated externally (Powers of Tau). Cannot be fixed in code alone.  
**Fix:** Run EIP-4844 compatible KZG ceremony and deploy the 96-byte compressed G₂[τ] point.

### ✅ MB-2: blob_to_kzg_commitment Real G1 MSM — FIXED (2026-06-27)
**File:** `zbx-da/src/commitment.rs`  
`blob_to_kzg_commitment` now computes `C = Σᵢ aᵢ · g1_srs[i]` (real BLS12-381 G1 MSM). `KzgSettings` extended with `g1_srs: Vec<G1Affine>` (4096 points). Devnet (τ=1): all entries = `G1Affine::generator()`. `load_from_ceremony_json` now also parses `g1_monomial`/`g1_lagrange`. `with_g1_srs()` builder method added. 4 tests: zero-blob→identity, constant-poly MSM correctness, wrong-size→identity, roundtrip commit+verify using the real commitment function.

### MB-3: AI Precompile Uses Stub Model Weights
**File:** `zbx-ai-precompile/src/model.rs:168`, `engine.rs:132`  
All 12 AI inference models use `stub_network()` — weights derived deterministically from `model_id` byte, not real trained models.  
**Fix:** Load real model weights from the DA layer or bundled binary.

### ✅ MB-4: Consensus VRF Verify — FIXED (2026-06-27)
**File:** `zbx-crypto/src/vrf.rs`  
`vrf_prove` now produces `output = keccak256(privkey ‖ input)` and signs `keccak256("zbx-vrf-v1\x00" ‖ input ‖ output)` with secp256k1 ECDSA, storing r‖s (64 bytes) in the proof field. `vrf_verify` recomputes the hash, tries both recovery ids (0 and 1), and accepts iff a recovered pubkey matches the expected `pub_key_bytes`. 7 unit tests added — prove→verify roundtrip, wrong pubkey, wrong input, tampered output, tampered proof bytes, determinism, distinct inputs. Unforgeable without the private key.

### ✅ MB-5: Whistleblower Bond Escrow — FIXED (2026-06-27)
**Files:** `zbx-staking/src/slashing_v2.rs`, `zbx-staking/src/persistence.rs`  
The `SlashingRegistryV2` struct no longer has a `pending_bonds` field — it was removed as part of the staking upgrade. Bond state is now fully durable via `EvidenceStore::put_bond` / `get_bond` / `delete_bond` / `list_bonds_for_record`, all backed by the `SlashingBonds` RocksDB column family. Bond entries are `bincode`-serialized `BondEntry` values keyed by `(record_id, reporter)`. Node restart cannot lose bond state. Module doc in `pipeline.rs` updated to reflect the fix.

### ✅ MB-6: BLS Zero Signature Fallback in build_tc — FIXED (2026-06-27)
**File:** `zbx-consensus/src/hotstuff2.rs`  
`build_tc` now returns `Option<TimeoutCertificate>` instead of `TimeoutCertificate`. On BLS aggregation failure it logs `error!` and returns `None` via `.ok()?` — no zeroed signature is ever constructed or propagated. `add_share` passes the `Option` directly to `on_timeout_share`, which already uses `if let Some(tc) = ...` — no call-site changes needed. Zeroed TC propagation is now impossible.

## Session Fix Log — 2026-06-27 (Pass-18)

### ✅ Fix-1: Governance Module Wire-Up
**File created:** `zbx-admin/src/governance.rs`  
**File updated:** `zbx-admin/src/lib.rs`  
The full governance engine that had been authored but left in `_archive/governance.rs` (dead code since genesis) is now registered as `pub mod governance` in `zbx-admin/src/lib.rs` and all types are re-exported at the crate root. Lifecycle: create → vote → finalize → timelock → execute → guardian veto. Double-vote prevention, quorum 10%, 7-day voting period, 2-day timelock. **11 unit tests** (new: `low_vote_count_leads_to_defeat`, `finalize_too_early_rejected`, `execute_before_timelock_rejected`, `update_total_staked_affects_quorum`).

### ✅ Fix-2: Bundler Relay — Real eth_sendRawTransaction
**File rewritten:** `zbx-bundler/src/relay.rs`  
**File updated:** `zbx-bundler/Cargo.toml` (added `reqwest 0.12`, `k256 0.13`)  
Previous implementation returned a deterministic SHA-256 hash and `block 1` without touching the network.  
New implementation:
- Inline RLP encoder (EIP-2718/1559 type-2 transaction — no external `rlp` crate).
- Full ABI encoder for `handleOps(UserOperation[],address)` selector `0x1fad948c` — correct head/tail layout for all 11 struct fields including 4 dynamic `bytes` tails.
- `eth_getTransactionCount` nonce fetch.
- `eth_gasPrice` + 2 Gwei tip → max_fee.
- `k256` secp256k1 EIP-1559 transaction signing.
- `eth_sendRawTransaction` submission.
- `eth_getTransactionReceipt` polling (60 × 2 s = 120 s deadline).
- Detects on-chain revert (`status = "0x0"`) → `BundlerError::BundleReverted`.
- Key from `ZBX_BUNDLER_PRIVKEY` env var or constructor arg.
- **7 unit tests** (selector, 32-byte alignment, determinism, RLP edge cases, EIP-1559 type byte, missing key error, valid key from field).

### ✅ Fix-3: Bundler Simulation — Real eth_call + ValidationResult Decode
**File rewritten:** `zbx-bundler/src/simulation.rs`  
Previous implementation returned `valid: true` for all structurally valid ops without any RPC call.  
New implementation follows ERC-4337 §6 simulation protocol:
- Encodes `simulateValidation(UserOperation)` calldata (selector `0xee219423`).
- Sends `eth_call` via `reqwest` JSON-RPC client.
- Extracts revert data from the JSON-RPC error `data` field (simulateValidation ALWAYS reverts intentionally).
- Decodes `ValidationResult` (selector `0xe0cff05f`) or `ValidationResultWithAggregation` (`0x00fa072b`): checks `sigFailed`, extracts `preOpGas`, `validAfter`, `validUntil`.
- Any non-ValidationResult revert → `BundlerError::SimulationRejected`.
- **8 unit tests** (selector, alignment, wrong selector, too-short payload, sigFailed=true, success decode, conflicts logic).

### ✅ Fix-4: Error Variants Added
**File updated:** `zbx-bundler/src/error.rs`  
Added: `MissingPrivKey`, `InvalidPrivKey`, `InclusionTimeout`, `BundleReverted`, `SimulationRejected`.
