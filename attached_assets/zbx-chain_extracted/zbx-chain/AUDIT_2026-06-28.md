## Fixes Applied — 2026-06-28 (This Audit Pass)

### 🔴 F1 — `contracts/BridgeMultisig.sol` — MS1 Tally Griefing (CRITICAL)

**Problem:** A compromised relayer could cast the first vote with garbage `(to, amount)` for a `zebvixSeq`, permanently locking the tally. Subsequent honest relayers receive `MismatchedTally`. User's bridged funds are permanently bricked with no recovery path.

**Fix:** Added `cancelTally(uint64 zebvixSeq) external onlyAdmin`:
- Resets `count`, `to`, `amount` to zero
- `votedBy[seq][addr]` remains — griefing relayer cannot re-vote
- Cannot cancel an already-executed (`executed == true`) tally
- Admin must then call `setRelayers` + wait `RELAYER_QUARANTINE` (86400s)

**Status:** ✅ FIXED

---

### 🔴 F2 — `crates/zbx-tx/src/signing.rs` — Cross-Network Replay (P0-T07) (HIGH)

**Problem:** Devnet and testnet share `chain_id = 8990`. EIP-155 protects on-chain transactions but NOT personal-sign messages (paymaster signatures, session keys, bridge relay proofs). A message signed on devnet is byte-identical to one valid on testnet.

**Fix:** Added network-scoped domain tags:
```rust
pub const SIGNING_DOMAIN_MAINNET: &[u8] = b"ZEBVIX_MAINNET_V1\x00";
pub const SIGNING_DOMAIN_TESTNET: &[u8] = b"ZEBVIX_TESTNET_V1\x00";
pub const SIGNING_DOMAIN_DEVNET:  &[u8] = b"ZEBVIX_DEVNET_V1\x00";
```
Canonical usage: `digest = keccak256(domain_tag || message)`  
Added `SigningContext::devnet()` factory + `network_domain()` accessor.

**Status:** ✅ FIXED

---

### 🔴 F3 — `tests/unit/crypto.rs` — Vacuous Stub Tests (HIGH)

**Problem:** Three test modules were complete stubs:
- `zbx_keccak256` local function returned `[0u8; 32]` → vector tests would FAIL if run
- `address_from_pubkey` compared string to itself (tautology)
- `sign_and_verify_roundtrip` → `assert!(true)` — never tested anything
- All merkle tests → `assert!(true)` — never tested anything

**Fix:** Full rewrite using real `zbx_crypto` APIs:
- `keccak_tests` — known Ethereum vectors (`keccak256("") = c5d246...`)
- `secp256k1_tests` — real `PrivKey::random()` → `sign()` → `recover_signer()` + rejection path
- `personal_sign` / `recover_personal_signer` roundtrip + cross-method rejection
- `merkle_tests` — real 4-leaf `MerkleTree`, `proof.verify()`, tampering scenarios

**Status:** ✅ FIXED

---

### 🔴 F4 — `tests/integration/aa_test.rs` — Paymaster Stub (HIGH)

**Problem:** `paymaster_sig_valid = true` hardcoded. ERC-4337 paymaster ECDSA validation was completely untested — test would pass even if validation was broken.

**Fix:** Real ECDSA roundtrip per ERC-4337 §6.1:
- `PrivKey::random()` → `personal_sign(digest, key)` → `recover_personal_signer`
- Happy path: recovered == paymaster_addr ✅
- Rejection path: attacker key → recovered ≠ paymaster_addr ✅

**Status:** ✅ FIXED

### 🔴 F7 — `config/testnet-genesis.json` — Single Validator (HIGH)

**Problem:** Genesis had 1 validator. HotStuff-BFT with n=1: `quorum = (2×1)/3 + 1 = 1`. Zero Byzantine fault tolerance — single node crash halts testnet permanently.

**Fix:** Added 3 more validators (4 total). Quorum now = 3, tolerates f=1 Byzantine failure.  
Added allocations for all 4 validators (gas money) + faucet address (10M ZBX).

```
Before: validators: ["0xad805b50..."]          quorum=1, f=0
After:  validators: ["0xad805b50...", "0x3C44...", "0x90F7...", "0x15d3..."]  quorum=3, f=1
```

**Status:** ✅ FIXED

---

### 🟡 F8 — `crates/zbx-bridge/Cargo.toml` — Missing Dev Dependencies (MEDIUM)

**Problem:** `[dev-dependencies]` section completely missing. `zbx_crypto::test_keys::test_keypair` helper (exists, gated by `feature = "testing"`) could not be used in bridge tests.

**Fix:** Added:
```toml
[dev-dependencies]
zbx-crypto = { path = "../zbx-crypto", features = ["testing"] }
```

**Status:** ✅ FIXED

### 🟡 F9 — `crates/zbx-bridge/src/relayer.rs:848` — Compile Error (MEDIUM)

**Problem:** `r.submit(req)` called with 1 arg; `BridgeRelayer::submit` requires 2 (`req, now: u64`). Compile error — test never ran in CI.

**Fix:** `r.submit(req, TS_NOW).expect("submit must succeed for valid request")`

**Status:** ✅ FIXED
